- :mod:`sklearn.linear_model` models that support `sample_weight` and
  internally use `linear_model._base._preprocess_data` or
  `linear_model._base._pre_fit` (e.g. :class:`linear_model.LinearRegression`,
  :class:`linear_model.Ridge`, :class:`linear_model.Lasso`,
  :class:`linear_model.ElasticNet`, :class:`linear_model.BayesianRidge`)
  now no longer make an unnecessary copy of dense `X, y` input during
  preprocessing when `copy_X=False` and `sample_weight` is provided.
  By :user:`Junteng Li <JasonLiJT>`
