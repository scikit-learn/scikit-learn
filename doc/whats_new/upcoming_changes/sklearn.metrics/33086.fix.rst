- :func:`metrics.accuracy_score`, :func:`metrics.hamming_loss`
  :func:`metrics.zero_one_loss`, :func:`metrics.matthews_corrcoef` and
  :func:`metrics.confusion_matrix` (when `labels` is not `None`) now
  raise an error for mixed string `y_true` and numeric `y_pred` inputs when
  inputs are list or numpy arrays not of `object` dtype.
  The above metrics, except for :func:`metrics.confusion_matrix`, will
  also raise an error for :term:`label indicator matrix` inputs of inconsistent size.
  By :user:`Lucy Liu <lucyleeow>`.
