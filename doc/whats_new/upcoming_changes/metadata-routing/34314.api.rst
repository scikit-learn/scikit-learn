- Metadata :term:`consumers <consumer>` that are not :class:`~base.BaseEstimator`
  subclasses (but inherit from :class:`~utils.metadata_routing._MetadataRequester`) can
  now customise class-level metadata requests by overriding
  :meth:`~utils.metadata_routing._MetadataRequester.__sklearn_build_class_level_metadata_request__`.
  :class:`~utils.metadata_routing.MethodMetadataRequest` and
  :func:`~utils.metadata_routing.get_class_level_metadata_request_values` are now public
  in :mod:`~utils.metadata_routing` for use in such overrides. See the :ref:`metadata
  routing developing guide
  <sphx_glr_auto_examples_miscellaneous_plot_metadata_routing.py>` for examples.
